Introducción Arquitectura Endpoints Detalles Clave

Proyecto ms-recetas

Una API RESTful robusta para una aplicación de recetas, con un enfoque en seguridad, escalabilidad e integración de servicios de terceros para pagos.

Frontend

App cliente desarrollada con React para una experiencia de usuario fluida en web y móvil.

Backend

Microservicio con Spring Boot que centraliza la lógica de negocio, autenticación y gestión de datos.

Base de Datos

PostgreSQL como sistema de persistencia, garantizando la integridad y consistencia de los datos.

Diagrama de Arquitectura de Software

Frontend

React — llamadas HTTPS a la API.

Backend

Spring Boot: Auth, endpoints, persistencia y pagos.

Storage & Pago

Postgres y proveedor de pagos (Flow) vía webhook.

Endpoints Principales de la API

POST /usuarios Crea un nuevo usuario con contraseña encriptada.
POST /usuarios/login Autentica y devuelve un token JWT.
POST /engagement/* Gestiona interacciones como comentarios, likes y ratings (protegido).
POST /flow/button Genera una orden de pago para el cliente.
POST /flow/webhook Recibe notificaciones de pago asíncronas de Flow.

Detalles Clave de Implementación

Seguridad y Autenticación

Uso de BCrypt para el hashing de contraseñas y JWT para la autorización basada en tokens. Los endpoints protegidos validan la propiedad de los recursos (owner check) para prevenir accesos no autorizados.

Integración de Pagos

El flujo de pagos es gestionado por un proveedor externo (Flow) de forma segura. El backend procesa los resultados a través de un webhook, manteniendo el sistema desacoplado y resiliente.

Operaciones y DevOps

Se implementa logging estructurado para facilitar el monitoreo y la depuración. Los entornos están separados, con herramientas de desarrollo (ej. /tools/hash) deshabilitadas en producción para mayor seguridad.
